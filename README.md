# Инструкция по Git

## Что такое Git?
Git - это система контроля версии. Она помогает сохранить историю изменении в проекте.

## Что такое репозиторий?
Репозиторий - это папка проекта, за которой следит Git и хранит историю изменении.

## Основные команды
### Навигация в терминале
- pwd - показать текущию директорию
- ls - показать файлы в директории
- ls -a - показать все файлы, в том числе и скрытые (например, .git, где Git хранит историю проекта)
- cd folder_name - перейти в папку
- cd ~ - перейти в домашнюю директорию
- cd .. - перейти на уровень выше

### Создание файлов и папок
- mkdir folder_name  - создать папку
- touch file_name - создать файл

### Git
- git init - инициализировать репозиторий
- git status - посмотреть состояние репозитория
- git add . - добавить все файлы в staging(промежуточная зона перед сохранением)
- git commit -m "message" - создать коммит
- git log - посмотреть историю коммитов

## Что такое коммит
Коммит — это сохранённое состояние проекта в определённый момент времени.

## Что такое GitHub
GitHub — это сайт, где можно хранить удалённые репозитории и работать с ними в команде.

## Как загрузить проект на GitHub
1. Создать локальную папку проекта
2. Выполнить `git init`
3. Создать файл README.md
4. Выполнить `git add .`
5. Выполнить `git commit -m "first commit"`
6. Создать репозиторий на GitHub
7. Связать локальный и удалённый репозитории
8. Выполнить `git push`

## Полезные команды для GitHub
- `git remote add origin <ссылка>` — привязать удалённый репозиторий
- `git branch -M main` — переименовать ветку в main
- `git push -u origin main` — отправить проект на GitHub
